Flask API - Test

Author: Yan Carvalho Borges

Technologies used:

Flask MongoDB (pymongo) JWT (pyjwt)

Usage:

Going to /auth page, an access token will be given.
I setted password in code because the intention was only make the authorization system to work, saving some time of coding another kind of document for user management

Pages:

  • /auth

    • Returns a access token based on users credential and Flask secret key. User is unset and password is 123

  • /transactions/insert

    • Takes a list of documents for inserting

    Token required!
  • /transactions/get

    • Return a list with all the documents stored

    Token required!
  • /transactions/patch

    • Takes a single document for patching data, filters based on Id field

    Token required!
  • /transactions/delete

    • Takes a single document with an Id value for deleting matching documents

    Token required!
  • /transactions/search

    • Recieves a document with DataHora and Valor field, and return any documents where values are greater or equal then the ones passed

    Token required!

File formatting

This is the schema setted for each transaction:

{ "DataHora": "timestamp", "Id": "string", "ContaInicial": "double", "ContaFinal": "double", "Valor": "double" }
